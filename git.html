3 集中型版本控制工具和分布式版本控制工具

3.1 集中型的版本控制工具

svn,cvs,vss
特点：
1. 实现大部分开发中对版本管理的需求
2. 结构很简单，上手容易
缺点：
1. 对中央仓库依赖严重，一旦损坏了，历史数据怎么恢复？
2. 程序员上传的都是完整版，如何追溯查询？
3. 系统正在上线运行，需要修改bug或增加几个不同的功能，持续几个月的时间。如何同时管理几个版本
4. 如果管理一个分布在世界上各地且互不相识的大型开发团队

3.2 分布式版本控制工具

github是一个git项目托管网站，主要提供基于git的版本托管服务。
